# Module 02: Virtual Twin Basics

This module explains virtual-twin-style thinking in plain language. It does not claim that this repository provides a live factory virtual twin.

## Learning Goals

- Explain the difference between CAD, digital twin, and virtual twin language.
- Understand why manufacturing stakeholders care about process and line context, not only geometry.
- Show how this repo can support virtual-twin-style conversations without overclaiming.

## Practical Framing

- CAD file: the part shape.
- Manufacturing context: the routing, work centers, inspection logic, and planning assumptions around the part.
- Virtual-twin-style review: testing scenarios and quality implications in a structured digital model before changing the floor.

## Demo Steps

1. Open the manufacturing context JSON.
2. Show how the routing and work-center assumptions become a deterministic flow report.
3. Show how inspection findings can be mapped back to operations and features.
4. Explain what is still missing versus a real, synchronized virtual twin platform.

## Discussion Questions

- Where would live machine data enter the story?
- Which teams would own the master routing and quality-gate definitions?
- How should a portfolio demo talk about virtual twins without overselling?
