# FreeCAD Automation v1.1.0 Release Checklist

## Merge Target

- Current merge branch: `integration/v1.1-release-candidate`
- Target branch for the release merge: `master`
- Create the `v1.1.0` tag from the release commit on `master`, not from a side branch.

## Version Confirmation

- Confirm [`package.json`](../../package.json) still reports `"version": "1.1.0"`.
- Confirm the release notes draft title and publish target still reference `v1.1.0`.
- Confirm the README release-surface links still point at the current support matrix, testing docs, config schema docs, output contract docs, and release notes draft.
- Confirm [`docs/releases/v1.1.0-final-audit.md`](./v1.1.0-final-audit.md) matches the branch head being merged.

## What CI Proves

- Hosted CI proves:
  - `npm run test:node:contract`
  - `npm run test:node:integration`
  - `npm run test:snapshots`
  - `npm run test:py`
- Hosted CI does not prove a live FreeCAD install or launch on any platform.
- Self-hosted macOS runtime smoke proves:
  - `fcad check-runtime`
  - `fcad create`
  - `fcad draw --bom`
  - `fcad inspect`
  - `fcad fem`
  - `fcad report`

## What Runtime Smoke Proves

- Runtime smoke proves the repository-owned macOS FreeCAD path works end to end for `check-runtime`, `create`, `draw --bom`, `inspect`, `fem`, and `report`.
- Runtime smoke does not prove Linux, Windows native, or WSL -> Windows runtime execution.
- Runtime smoke still does not elevate `tolerance` to a repository-owned CI claim.

## What Is Still Compatibility-Only

- Linux local with FreeCAD installed
- Windows native + FreeCAD 1.1
- WSL -> Windows FreeCAD
- Any hosted CI environment that does not install or launch FreeCAD

## Repo Metadata

- Confirm the GitHub repository description matches the v1.1.0 product identity.
- Confirm topics cover the current public surface (`freecad`, CAD, TechDraw, FEM, manufacturing, automation).
- Leave the GitHub homepage blank unless a separate public docs or product URL exists.

## Final Validation Checklist

Run these against the release-candidate head you intend to merge. If any command cannot run because of the local machine, mark it explicitly as `BLOCKED BY ENVIRONMENT` instead of treating it as passed.

- `npm ci`
- `node bin/fcad.js --help`
- `npm run test:node:contract`
- `npm run test:node:integration`
- `npm run test:snapshots`
- `npm run test:py`
- `node bin/fcad.js check-runtime`
- `node bin/fcad.js check-runtime --json`
- `node bin/fcad.js validate-config configs/examples/ks_bracket.toml --strict`
- `node bin/fcad.js validate-config configs/examples/controller_housing_eol.toml --strict`
- `npm run test:runtime-smoke`

Capture or link the self-hosted runtime smoke artifacts when that lane is part of the sign-off evidence.

## Tag And Release Steps

1. Confirm the release-candidate head contains the final audit and any release-surface doc fixes.
2. Merge `integration/v1.1-release-candidate` into `master`.
3. Pull `master` locally and confirm the release docs are present on the exact merge commit.
4. Confirm the hosted `Automation CI (hosted fast lanes)` status is green for the merge commit, especially the Python lane if it was `BLOCKED BY ENVIRONMENT` locally.
5. Confirm the latest relevant `FreeCAD Runtime Smoke (self-hosted macOS)` run or local self-hosted rerun matches the merge commit and includes `fem`.
6. Create and push tag `v1.1.0` from `master`.
7. Create the GitHub release using [`docs/releases/v1.1.0-draft.md`](./v1.1.0-draft.md) as the release body and [`docs/releases/v1.1.0-final-audit.md`](./v1.1.0-final-audit.md) as the operator audit attachment/reference.
