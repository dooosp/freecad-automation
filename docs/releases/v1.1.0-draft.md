# FreeCAD Automation v1.1.0 Draft

Recommended first public tag: `v1.1.0`

## Positioning

FreeCAD Automation is a FreeCAD-backed automation pipeline for CAD generation, TechDraw drawings, inspection, FEM, reporting, and manufacturing-review artifacts.

This release should present the repository as:

- a runtime-backed CLI for CAD, TechDraw, FEM, tolerance, inspection, and reporting
- a plain-Python/Node manufacturing-review layer for DFM, process planning, readiness reporting, stabilization review, and review-pack artifacts

## Verified Platform And Runtime Scope

- Verified maintainer path: macOS with FreeCAD 1.1.x app bundle
- Verified commands on that path: `check-runtime`, `create`, `draw --bom`, `inspect`, `fem`, `report`
- Windows native and WSL -> Windows FreeCAD remain compatibility paths driven by explicit `FREECAD_*` overrides
- Linux path detection exists, but the repository does not currently claim live runtime smoke coverage there

## Automated Smoke-Test Scope

Hosted CI currently covers:

- Node runtime detection/path/invocation contract tests on `ubuntu-24.04` and `macos-14`
- Python unit and CLI tests on `ubuntu-24.04` with Python 3.11

Hosted CI does not currently install or launch FreeCAD.

Runtime smoke coverage is prepared through:

- `npm run smoke:runtime`
- `.github/workflows/freecad-runtime-smoke.yml`

The current smoke path exercises:

- `fcad check-runtime`
- `fcad create`
- `fcad draw --bom`
- `fcad inspect`
- `fcad fem`
- `fcad report`

That workflow requires a self-hosted macOS runner labeled `freecad` with FreeCAD 1.1 installed.

## Known Limitations

- Public support should not be described as fully cross-platform end-to-end validated.
- Windows/WSL and Linux runtime paths are documented, but not yet backed by repository-owned runtime smoke coverage.
- `analyze-part` is conditional: it can run without FreeCAD when context already includes model metadata, but it will use FreeCAD for live model inspection or STEP feature detection.
- `report` is still a runtime-backed path because macOS fallback uses the bundled FreeCAD Python executable.
- The manufacturing-review layer produces heuristic engineering artifacts and still requires engineering validation.

## What Is Still Manual

- Verifying any platform/runtime path outside macOS + FreeCAD 1.1
- Validating the self-hosted FreeCAD smoke workflow on the repository’s actual runner fleet
- Human review of release packaging, publishing, signing, and any downstream release distribution steps

## Notes For Release Summary

- First impression no longer implies WSL-first or Windows-first usage
- Public messaging centers the repository on a FreeCAD-backed automation pipeline, not a bridge workaround
- Runtime boundary between FreeCAD-backed and plain-Python commands is documented explicitly
- `check-runtime` is the first troubleshooting entrypoint in docs and CLI help
- Release should describe hosted CI and runtime smoke honestly, without implying hosted FreeCAD execution
